## 🐛 发现的问题（非阻塞）

### 1. 远程 backend 健康检查失败（unhealthy）

**原因：** 容器内缺 `curl`，健康检查脚本执行失败（exit code -1），不是服务本身问题。
**影响：** 仅影响 docker 状态显示，实际 `/health` 接口返回 200，服务正常。
**建议修复：** 在 Dockerfile 安装 curl，或改用 `python -c "import urllib.request; ..."` 作为健康检查命令。

### 2. 数据库存在重复持仓数据

**现象：**
- `positions` 表中 "沪深300ETF" 有 3 条记录（id 8/15/17）
- `assets` 表中 "[持仓] 沪深300ETF" 有 2 条记录（id 20/22）
- `positions` 表中 "验收测试基金" 有 2 条记录（id 21 + 1 条验收残留）

**影响：** 资产总值会被重复计算，导致总资产虚高。
**建议修复：** 清理重复数据 + 增加唯一约束（name + symbol + account 组合唯一）。

### 3. 账户余额 vs 总资产来源不一致

**现象：**
- 首页"总资产"来自 `assets` 表（237938 元）
- 账户余额总和来自 `accounts` 表（6924 元本地 / 4412 元远程）
- 两者差异很大，因为 `assets` 包含投资持仓、储蓄卡等，`accounts` 只包含交易账户

**影响：** 用户可能困惑"总资产 23 万"但"账户余额才 6 千"。
**建议：** 在首页增加说明文字，区分"交易账户余额"和"总资产"的口径。

### 4. 远程服务器 backend 镜像与本地不一致

**现象：**
- 远程 `/investments/positions` 返回 404（旧路由）
- 本地 `/investments/positions` 也 404，正确路由是 `/positions`
- 远程交易 ID 已到 4888，本地只有 59（数据完全隔离）

**影响：** 远程是生产环境，数据真实；本地是开发环境，数据是测试数据。
**建议：** 明确区分开发/生产环境，避免混淆。
