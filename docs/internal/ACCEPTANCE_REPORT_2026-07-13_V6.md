## ⚠️ 发现的问题

### 🔴 P0：前后端数据不一致

**现象：**
- 前端 UI 显示：总资产 ¥238,287.74，35+ 条资产
- 后端 API 返回：总资产 ¥150,500.00，7 条资产
- 差异：¥87,787.74（约 58%）

**根因分析：**
- Backend 容器 2 小时前重启（可能重建了 DB）
- Frontend 容器 9 小时前启动，缓存了旧数据
- Hydration mismatch 错误证实了 SSR/CSR 数据不一致

**影响：** 用户看到的资产数据与实际存储不一致，核心功能可信度受损

**建议修复：**
1. 前端每次页面加载时强制从 API 刷新数据（不使用缓存）
2. 添加 API 数据与前端状态的校验机制
3. 考虑在 backend 启动时做数据完整性检查

### 🟡 P1：Hydration Mismatch
- 首页加载时出现 `Hydration completed but contains mismatches`
- 非致命，但影响 SSR 性能和 SEO
- 可能与数据不一致问题相关

### 🟡 P2：测试数据残留
- 资产页曾出现两个"验收测试基金"条目（已手动清理）
- 建议：测试结束后自动清理，或使用事务回滚
