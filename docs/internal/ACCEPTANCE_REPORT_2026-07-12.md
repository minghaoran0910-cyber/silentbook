## 二、P0 Bug 详情

### Bug 1: 交易API路径不匹配

**现象：**
- 交易页显示骨架屏，列表无法加载
- 浏览器控制台报错：`GET /api/transactions?limit=10` → 500/404

**根因：**
- 前端配置：`NUXT_PUBLIC_API_BASE=/api`
- 后端路由：`/transactions`（无 `/api` 前缀）
- 客户端请求 `/api/transactions` → 后端返回 404

**影响：**
- 交易记录完全无法查看
- 影响18:00正式使用

**解决方案（三选一）：**

**方案A：后端添加API前缀（推荐）**
```python
# backend/app/main.py
app = FastAPI()
api_router = APIRouter(prefix="/api")  # 添加这行

# 所有路由改为：
@api_router.get("/transactions")
async def get_transactions():
    ...

app.include_router(api_router)
```

**方案B：前端配置改空**
```bash
# .env 文件
NUXT_PUBLIC_API_BASE=
```

**方案C：Nginx反向代理**
```nginx
location /api/ {
    proxy_pass http://backend:8000/;
}
```

**建议：** 方案A最规范，方案B最快（改一行配置重启即可）

### Bug 3: 认证问题（P1）

**现象：**
- 多个API返回401（登录已过期）
- 但首页SSR数据正常显示

**根因：**
- SSR时直接访问后端（`NUXT_SSR_API_BASE=http://backend:8000`），不需要认证
- 客户端请求经过前端代理，需要认证token
- 浏览器没有登录态

**影响：**
- 客户端导航后部分功能受限
- 但SSR页面首次加载正常

**解决方案：**
- 检查认证逻辑，确保客户端请求也带token
- 或临时关闭认证（开发模式）

---

### Bug 4: Hydration Mismatch（P2）

**现象：**
- 控制台报错：`Hydration completed but contains mismatches.`

**根因：**
- SSR渲染的数据和客户端初始数据不一致
- 可能是因为API路径问题导致客户端数据为空

**影响：**
- 轻微，不影响功能
- 但可能导致页面闪烁

**解决方案：**
- 修复API路径后可能自动解决
