## 正文

各位 V 友好，分享一个我做的开源项目 SilentBook。

### 解决什么问题

我一直觉得记账 App 有个矛盾：**手动记账太累，自动记账不准**。

银行和支付平台的通知其实已经包含了所有交易信息，但没人帮你结构化。市面上的自动记账要么依赖 App 权限，要么是云端抓取，隐私和便利总得选一个。

SilentBook 的思路是：**直接解析通知短信/推送，本地处理，AI 分析**。

### 核心能力

1. **通知解析** — 支持招行、工行、建行、支付宝、微信支付五大平台，粘贴通知文本即可自动识别金额、时间、商户、分类

2. **AI Agent 协同** — 不只是一个 AI，是多个 Agent 各司其职：
   - 消费分析 Agent：看消费结构、异常支出
   - 投资分析 Agent：看持仓收益、风险评估
   - 综合建议 Agent：整合多方视角，给财务规划建议

3. **数据可视化** — 消费趋势图、资产变化曲线、分类饼图、收支对比，日/周/月/年报

4. **预算管理** — 分类预算 + 超支预警

5. **三种部署模式** — 本地（全在本地）、混合（服务器前端+本地数据）、云端（全云端），按隐私需求选

### 技术栈

- 前端：Vue 3 + Nuxt 3，深色主题
- 后端：FastAPI + PostgreSQL
- AI Agent：可对接任意 LLM API
- 部署：Docker Compose，一行命令启动

### 为什么开源

记账是个高度个性化的需求，每个人关心的维度不一样。与其做一个封闭产品，不如把骨架开源，让每个人按自己的需求定制。MIT 协议，随便用。
