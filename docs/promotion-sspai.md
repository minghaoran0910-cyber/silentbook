# SilentBook：让通知短信变成你的财务管家

> 财务自由，不是终点，是每一步的选择。

## 一、记账的永恒矛盾

你一定经历过这样的场景：

月底打开银行 App，看着「本月支出 ¥4,832」发呆。这钱花哪了？似乎也没买什么大件，怎么就花了这么多？你想搞清楚，于是下载了某个记账 App，坚持了三天，第四天忘了记，第五天补不上，第六天就放弃了。

**手动记账的死穴在于：它要求你每一次消费都做一次有意识的动作。** 而人的天性是，花钱的时候不想记账，记账的时候不想花钱。

那自动记账呢？市面上的方案要么要你开通银行 API 授权，要么在手机上装一个辅助功能插件读取通知——前者门槛高，后者让人不安。

但我后来意识到一件事：**你收到的每一条支付通知，已经包含了这笔交易的全部信息。**

> 招商银行：您尾号1234的储蓄卡于12月25日14:32在星巴克消费人民币38.50元，余额12,560.30元。

金额、时间、地点、余额，全在这条短信里。缺的只是一个帮你结构化、分类、分析的工具。

SilentBook 就是这个工具。

## 二、SilentBook 是什么

SilentBook 是一个开源的全自动无感记账系统，核心理念是：**你不需要改变任何习惯，你的通知短信已经在告诉你一切。**

它做三件事：

1. **解析通知** — 粘贴或自动接收银行/支付宝/微信的通知文本，自动提取金额、时间、商户、分类
2. **AI 分析** — 多个 AI Agent 各司其职，分别分析消费结构、投资收益、给出综合建议
3. **可视化** — 趋势图、分类饼图、收支对比，日/周/月/年报一目了然

目前支持五大通知来源：招商银行、工商银行、建设银行、支付宝、微信支付。

## 三、为什么不止用一个 AI

市面上大部分 AI 记账工具的逻辑是：把所有数据扔给一个 AI，让它给你一段分析报告。听起来合理，但实际体验是——**什么都说了，等于什么都没说。**

消费和投资是两套完全不同的分析框架。看消费的人关注「钱花在了哪里、有没有浪费」；看投资的人关注「持仓结构对不对、风险敞口大不大」。让一个 AI 同时干两件事，结果是两边都浅尝辄止。

SilentBook 的做法是把分析拆成多个 Agent：

- **消费分析 Agent** — 专注消费结构、异常支出、分类占比、优化建议
- **投资分析 Agent** — 专注持仓收益、风险评估、资产配置
- **综合建议 Agent** — 整合两方视角，给出财务规划建议

每个 Agent 独立工作，各看各的维度，最后汇总。你可以只开一个，也可以全开。

## 四、隐私优先的三种部署模式

财务数据是最高密级的个人信息。SilentBook 提供三种部署模式，按你的隐私偏好选择：

### 本地模式（最私密）
所有服务跑在你自己的电脑上，数据不出门。适合完全不想把财务数据放到任何服务器上的人。

### 混合模式（推荐）
服务器只跑前端和后端，数据库和 AI Agent 在本地。你的原始数据始终在自己机器上，但可以在外面访问到展示层。

### 云端模式（最方便）
全部部署在云服务器上，多设备随时访问。适合需要在手机、平板、电脑之间无缝切换的人。

三种模式都是 Docker Compose 一行命令启动，切换模式只需换一个配置文件。

## 五、设计理念：安静、克制、不焦虑

很多记账 App 的设计逻辑是制造焦虑——打开就是红色的「超预算」、推送通知「你这个月已经花了 XXX」、排行榜「同龄人平均花多少」。这些设计或许能提高打开率，但对真正想理清财务的人来说，是噪音。

SilentBook 的设计原则是：

- **深色主题，电影质感** — 不刺眼，不焦虑
- **信息密度可控** — 你需要看的时候才看，不被推送绑架
- **无社交、无排行、无比较** — 你的财务是你自己的事
- **AI 分析有节制** — 有异常才提醒，不是每天给你写一篇小作文

## 六、技术实现

如果你是开发者，这里是你关心的部分：

| 模块 | 技术选型 |
|------|---------|
| 前端 | Vue 3 + Nuxt 3，响应式 + PWA |
| 后端 | FastAPI (Python)，JWT 认证，API 限流 |
| 数据库 | PostgreSQL |
| 通知解析 | 正则 + NLP 混合方案，5 大平台 38 个测试用例 |
| AI Agent | 可对接任意 LLM API，支持 OpenClaw 多 Agent 协同 |
| 部署 | Docker Compose，三种模式 |

项目结构清晰，前后端分离，每个模块独立 Docker 容器。想替换某个模块（比如用 React 替换 Vue，或用 Go 替换 Python 后端），只需改对应容器。

## 七、谁适合用

- **不想手动记账但想搞清楚钱花哪了的人** — 通知解析帮你自动记录
- **有投资习惯想跟踪整体资产的人** — 资产负债表 + 投资分析
- **注重财务隐私的人** — 本地部署，数据不出门
- **想定制化分析逻辑的人** — 开源代码，Agent 可自定义
- **想要多 Agent 协同分析的人** — 消费、投资、综合建议各管一摊

## 八、快速开始

```bash
git clone https://github.com/minghaoran0910-cyber/silentbook.git
cd silentbook
cp .env.example .env
# 编辑 .env，填入你的 API Key
docker-compose up -d
```

访问 `http://localhost:3000`，开始用。

详细文档见 GitHub 仓库 README。

## 九、为什么开源

记账是高度个性化的需求。每个人关心的维度不一样——有人在意餐饮占比，有人在意投资收益，有人只看月度总账。

做封闭产品意味着我要替所有人做选择，这不对。把骨架开源，让每个人按自己的需求定制，才是正确的方式。
