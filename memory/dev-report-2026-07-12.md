# SilentBook V2 开发报告 — 2026-07-12

## 一、PRD P0 Bug 验证

### 结论：PRD 描述的 P0 Bug 不存在

**PRD 声称**：前端请求 /api/transactions，后端返回 404，因为后端路由没有 /api 前缀。

**实际情况**：
- nginx 配置 `location /api/` → `proxy_pass http://127.0.0.1:8000/`（自动 strip /api/）
- 浏览器请求 `/api/transactions` → nginx 转发 → backend:8000/transactions ✅
- `NUXT_PUBLIC_API_BASE=/api` 是**正确的配置**，去掉反而会破坏路由
- 交易页和投资页都能正常加载数据

**如果按 PRD 方案去掉 /api 前缀**：
- 浏览器请求 `/transactions` → nginx 路由到前端 3000 → Nuxt 当成页面路由 → 404
- 整个 API 链路会全部断掉

## 二、发现并修复的真实 Bug

### Bug 1: 首页净资产计算公式错误 ✅ 已修复

**问题**：首页净资产 = ¥136,936.50，但资产页净资产 = ¥133,224.30

**根因**：后端 `backend/app/main.py` 第 541 行：
```python
net_assets = (total_income - total_expenses) + total_assets - total_liabilities
```
把历史收支又加了一遍到净资产里（交易已体现在资产值中，重复计算）

**修复**：
```python
net_assets = total_assets - total_liabilities
```

**验证**：修复后 net_assets = 137,500 - 4,275.70 = 133,224.30 ✅

**Commit**: `6840328` — fix: 修复首页净资产计算公式
